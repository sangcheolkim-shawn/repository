Journey 1 — 신규 콘텐츠 등록

  • 대상: 미등록 콘텐츠
  • 단계: 11단계 (풀코스)
  • 양식 정의 → 커리큘럼 설계
  • 콘텐츠 제작 → 검수
  • 과목코드 연결 → 배포
  • 버전 관리: 없음 (최초)
  • 핵심: 전체 파이프라인 실행

Journey 2 — 수정 (사용 전)

  • 대상: 등록 완료, 미배포
  • 단계: 7단계
  • 직접 수정 (덮어쓰기)
  • 재검수 → 연결 확인 → 배포
  • 버전 관리: 없음
  • 핵심: 원본 직접 수정, 경량 플로우

Journey 3 — 수정 (사용 중)

  • 대상: 운영 중 (배포 완료)
  • 단계: 9단계 + 롤백
  • 개정(revision) 생성 → 수정
  • 검수 → 재배포 + 롤백
  • 버전 관리: v1.0 → v1.1
  • 핵심: 원본 보호 + 버전 관리

Journey 3 — 콘텐츠 수정, 사용 중 9단계 + 롤백
운영 중 콘텐츠 개정(Revision) 플로우
배포 완료(live) 상태 → 개정본 생성 → 수정 → 검수 → 재배포 + 롤백
진입 조건: 콘텐츠가 운영 중(live) 상태 — 이미 서비스에 연결되어 사용 중
핵심 원칙: 운영 중인 원본(v1.0)은 수정 과정에서 절대 변경되지 않음
버전 관리: v1.0(운영 유지) → v1.1(개정본) 자동 버전 관리 + 롤백 지원

원칙설명적용 예시
최소 권한 원칙 각 역할에 업무 수행에 필요한 최소한의 권한만 부여 CW는 OWN 데이터만, QA는 검수 화면만 CRUD
직무 분리 콘텐츠 작성자(CW)와 검수자(QA)는 반드시 분리. 자기 작업을 자기가 검수할 수 없음 CW → 검수 ✕, QA → 콘텐츠 수정 ✕
OWN 데이터 격리 CW는 본인에게 배정된 작업만 조회/수정 가능. 타인 작업에 접근 불가 작업 진행 보드 OWN-R, 콘텐츠 등록 OWN-CRU
SA 마스터 권한 시스템 관리자(SA)는 전체 화면에 CRUD 마스터 권한을 보유. 콘텐츠 마스터(CM)는 콘텐츠 워크플로우 전담 SA: 전체 12개 화면 CRUD
