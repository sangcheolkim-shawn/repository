버전업 배포 — 화면 흐름

콘텐츠 배포와 서비스 반영(코드 연결)을 분리한 안전 배포 모델.
각 화면(메뉴)에서의 사용자 동선과 신규 기능을 풀스크린 레이아웃으로 설명합니다.
* 코드 승계 기능은 6/16 PI 회의에서 제외 결정 (구현 에너지 절감)

2026-06-16 회의 자료| 플랫폼기획팀 김상철| Release v1.5

배경: 지난 회의 교훈 → 이번 구현

❌ 회의에서 드러난 문제
1. 배포 = 서비스 영향? 혼동
콘텐츠 배포와 서비스 반영의 경계가 불명확하여, 담당자가 배포를 주저하거나 잘못된 시점에 코드를 연결하는 상황 발생
2. 코드 연결 시 안전장치 부재
미배포 콘텐츠가 있는 상태에서 과목코드가 연결되면 불완전한 콘텐츠가 서비스에 노출될 위험
✅ 이번 구현으로 해결
1. 안전 배너 도입 배포 관리
"콘텐츠 배포는 서비스에 직접 영향을 주지 않습니다" 메시지를 배포 대기 탭 상단에 상시 표시하여 불안감 해소
2. 코드 연결 가드레일 과목코드 연결
코드 연결 시 미배포 콘텐츠 경고 + "서비스에 즉시 반영됩니다" 확인 다이얼로그로 실수 방지
변경 관리 프로토콜 (L24, 2026-06-10 PI 회의 이후 수립)
이전 (문제) 이번 (해결) 구현 위치
배포 시 서비스 영향 여부 불명확 → 안전 배너로 "영향 없음" 명시 배포 관리 > 대기 탭 상단
코드 연결 수 파악 불가 → 카드에 "코드연결 N개" 뱃지 표시 배포 카드 하단 (bound-badge)
미배포 콘텐츠에 코드 연결 가능 → 가드레일: 미배포 경고 + 확인 필수 과목코드 연결 > "+ 연결" 클릭 시
배포/연결 분리 원칙 미인지 → "서비스에 즉시 반영됩니다" 경고문 코드 연결 가드레일 confirm 내

핵심 개념

🛡

콘텐츠 배포 = 안전

콘텐츠를 배포해도 서비스에 직접 영향 없음. 과목코드 연결 전까지 서비스 미노출.

🔗

과목코드 연결

배포된 콘텐츠에 과목코드를 연결하여 서비스에 반영. 과목코드 연결 메뉴에서 직접 수행.

⚠️

코드 연결 가드레일

과목코드 연결 시 사전 검증 수행. 미배포 콘텐츠가 있으면 경고 표시.

🛡

안전 모델 핵심

콘텐츠 배포와 서비스 반영(코드 연결)이 완전히 분리.
콘텐츠를 배포해도 과목코드 연결 전까지 서비스에 노출되지 않습니다.

#단계메뉴 위치화면 요소서비스 영향
1 배포 대기 확인 🚀 배포 관리 안전 배너 + 커리큘럼 카드 + 코드연결 뱃지 🟢 없음
2 과목코드 연결 🔗 과목코드 연결 연결된 코드 목록 + 추가 연결 가능 코드 🟡 주의
3 가드레일 검증 🔗 과목코드 연결 (confirm) 미배포 경고 + 기존 코드 확인 + "즉시 반영" 안내 🔴 직접 영향
✨ 핵심 포인트: Step 1은 배포 관리 메뉴에서 안전하게 진행. Step 2~3은 과목코드 연결 메뉴에서 서비스 반영 작업 수행. 코드 연결 시 가드레일이 반드시 동작합니다.
