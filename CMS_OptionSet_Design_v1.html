1. 문제 정의

개발팀 피드백: "뉴드림스 입력양식은 과목×유형 수만큼 생성할 수밖에 없을 것 같다"

핵심 충돌

CMS 2.0 아키텍처의 핵심 원칙 "양식은 과목 독립"과, 현실의 "select 필드 옵션이 과목마다 다르다"가 충돌합니다.

❌ 양식을 과목별 복제하면

  • 29개 과목 × 9개 양식 유형 = 261개 양식
  • 필드 1개 추가 시 261개 일괄 수정
  • "과목 독립" 아키텍처 원칙 붕괴
  • 운영자 관리 부담 폭증

✅ 옵션값만 분리하면

  • 양식 9개 유지 + 옵션 그룹 2~3개 추가
  • 필드 변경 시 1곳만 수정
  • 아키텍처 원칙 100% 유지
  • 운영자는 옵션값만 관리

4. 권장 방안: 옵션 세트 분리

양식 구조는 공유, 옵션값만 분기 — "Structure Sharing, Value Branching"

4.1 개념 모델

양식 (Form)
→
필드 정의 (FieldDef)
→
옵션 그룹 참조
→
런타임: subject로 세트 선택
→
드롭다운 옵션 렌더링

4.2 데이터 흐름 예시

시나리오: "수학 기초 과정" 커리큘럼에서 "정답·해설 양식"의 콘텐츠를 등록할 때

1. 편집기 진입 → 커리큘럼의 subject = 'M' 확인
2. 양식의 area 필드 → optionGroupId = 'OG-AREA' 참조
3. OG-AREA에서 scopeValue = 'M' 세트 검색
4. 옵션 세트 ['대수','기하','통계','해석'] 로드 → 드롭다운에 표시
5. 사용자는 "기하" 선택 → value = '기하' 저장

4.3 폴백 전략

  • 매칭 세트 존재 → 해당 세트의 옵션값 사용
  • 매칭 세트 없음 → is_default = true 세트 사용
  • 기본 세트도 없음 → LT_FIELD_POOL의 options 배열 폴백 (하위 호환)

5.2 DDL

-- 옵션 그룹: 과목별 분기가 필요한 select 필드 선언
CREATE TABLE ncm_field_option_groups (
    id           SERIAL PRIMARY KEY,
    field_key    VARCHAR(50)  NOT NULL,              -- LT_FIELD_POOL key
    group_name   VARCHAR(100) NOT NULL,              -- 표시명 (예: '학습 영역')
    scope_type   VARCHAR(20)  NOT NULL DEFAULT 'subject',  -- subject | grade | custom
    description  TEXT,
    created_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    UNIQUE(field_key)
);

-- 옵션 세트: scope_type 기준값별 옵션 컨테이너
CREATE TABLE ncm_field_option_sets (
    id           SERIAL PRIMARY KEY,
    group_id     INT          NOT NULL REFERENCES ncm_field_option_groups(id) ON DELETE CASCADE,
    scope_value  VARCHAR(50)  NOT NULL,              -- 'M','K','E'... 또는 '_default'
    set_label    VARCHAR(100),                       -- '수학 영역 옵션'
    is_default   BOOLEAN      NOT NULL DEFAULT FALSE,
    created_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    UNIQUE(group_id, scope_value)
);

-- 옵션 값: 실제 드롭다운 항목
CREATE TABLE ncm_field_option_values (
    id           SERIAL PRIMARY KEY,
    set_id       INT          NOT NULL REFERENCES ncm_field_option_sets(id) ON DELETE CASCADE,
    value        VARCHAR(200) NOT NULL,              -- 저장되는 값
    label        VARCHAR(200),                       -- 표시명 (null이면 value 그대로)
    sort_order   INT          NOT NULL DEFAULT 0,
    is_active    BOOLEAN      NOT NULL DEFAULT TRUE
);

7.2 런타임 해석 헬퍼

/**
 * 과목 컨텍스트에 따른 select 옵션 해석
 * @param {string} fieldKey  - LT_FIELD_POOL의 필드 key (예: 'area')
 * @param {string} scopeVal  - 분기 기준값 (예: 'M' = 수학)
 * @returns {string[]}       - 옵션 배열
 */
function _resolveFieldOptions(fieldKey, scopeVal) {
  var group = (OPTION_GROUPS || []).find(function(g) {
    return g.fieldKey === fieldKey;
  });
  if (!group) {
    // 옵션 그룹 없으면 LT_FIELD_POOL 기본값 폴백
    var pf = LT_FIELD_POOL.find(function(f) { return f.key === fieldKey; });
    return pf ? (pf.options || []) : [];
  }
  // 1순위: 정확 매칭 세트
  var set = group.sets.find(function(s) { return s.scopeValue === scopeVal; });
  // 2순위: 기본 세트
  if (!set) set = group.sets.find(function(s) { return s.scopeValue === '_default'; });
  return set ? set.values : [];
}

7.4 편집기 select 렌더링 변경

// 콘텐츠 편집기에서 select 필드 렌더링 시
// 변경 전: field.options를 직접 사용
// 변경 후: _resolveFieldOptions()로 과목별 옵션 해석

var curSubject = currentCurriculum.subject;  // 'M', 'K', 'E' 등

if (field.type === 'select') {
  // 옵션 그룹 참조 여부 확인
  var opts = field.optionGroupId
    ? _resolveFieldOptions(field.key, curSubject)
    : (field.options || []);

  p += '<select>';
  opts.forEach(function(opt) {
    p += '<option>' + opt + '</option>';
  });
  p += '</select>';
}

역할변경점영향도
