1. 종합 요약

CMS V9.9 프로토타입은 디자인·퍼블리싱 인력 없이 단일 HTML로 13개 메뉴를 구현한 상태입니다. 전체 완성도는 약 75%이며, 핵심 콘텐츠 워크플로우(작업 진행 → 설정 → 등록 → 검수)는 실제 운영 가능 수준으로 잘 갖춰져 있습니다.

잘 된 점

Apple HIG 기반 디자인 시스템(20개 컬러 토큰, SF Pro + Pretendard)이 일관되게 적용되어 있고, CSS 변수 체계가 우수합니다. 커리큘럼 3-Screen 패턴, 콘텐츠 등록 3패널 구조, 칸반보드 + 리스트 전환 등 정보 구조 설계가 탄탄합니다. 수신 동기화 4-state 모델(matched/cur-ahead/recv-ahead/diverged)과 2단계 안전 모델(배포 = safe → 코드 연결 = gate) 설계 사상도 적절합니다.

주요 우려

가장 큰 위험은 배포·롤백 과정의 안전장치 부족입니다. 현재 롤백이 검수 없이 confirm 한 번으로 실행되고, 미배포 콘텐츠에서도 코드 연결이 가능하며, 배포 실행과 스케줄 삭제에 추가 확인이 없습니다. 사용자가 많은 교육 서비스 특성상 휴먼 에러 한 번이 서비스 전체에 영향을 주기 때문에, 이 영역의 보강이 최우선입니다.

또한 로그인/인증 시스템과 역할 기반 접근 제어(RBAC)가 완전히 부재하여, 실제 운영 전에 반드시 구현이 필요합니다.

전체 진행률
기능 구현 75% · 안전장치 보강 필요 · 인증/권한 미구현

2. 메뉴별 완성도 평가

# 메뉴 완성도 구현된 기능 미비 사항
1 작업 진행 보드
workflow		 완성 칸반/리스트 전환, 다축 필터(과목/학년/상태/담당자), phase 드래그 이동, 바로가기 링크 칸반 카드 "바로가기" 라벨이 마지막 phase에서도 동일하여 모호
2 작업 설정
work-config		 완성 카드 그리드, 인라인 담당자 배정, 범위 pill 표시, 상태 필터 작업 범위 중복 체크 없음 (동일 커리큘럼을 두 작업에 배정 가능)
3 콘텐츠 등록
content-register		 완성 3패널 구조, 커리큘럼/파일 우선 2경로, 에디터, 메타 입력, CF 매핑 트리 강제 게시 이중확인 부족, 메타 입력 값 STATE 미반영, CF 매핑 검색 debounce 없음
4 검수 관리
review		 완성 체크리스트 검수, 반려 사유 입력, 검수 노트, 항목별 피드백 일괄 검수 요청 시 검수자 하드코딩 ("김검수(QA팀)"), 선택 모달 없음
5 배포 관리
deploy		 부분 3탭(대기/진행/완료), 안전 배너, 스케줄 배포 롤백 검수 없음, 배포 실행 추가 확인 없음, 스케줄 삭제 무확인, 코드 연결 뷰 직접 링크 없음
6 과목코드 연결
service-bind		 완성 바인딩 UI, 가드레일, 경고 다이얼로그, 해제 확인 미배포 콘텐츠에서도 코드 연결 가능 (불완전 콘텐츠 서비스 노출 위험)
7 커리큘럼 수신
cur-receive		 완성 동기화 4-state, 편집버전 생성, diff 비교, 영향분석 "반려"와 "수신 변경 무시"가 동일 함수 호출 — 라벨 혼동. 편집버전 생성 후 부분 갱신 혼란
8 서비스 콘텐츠
service-content		 부분 콘텐츠 상세, 버전 이력 조회 롤백 기능이 스텁(alert만 표시), 실제 롤백 로직 미구현
9 양식 관리
form-manage		 완성 필드 풀, 양식 에디터, 삭제 가드(사용 중 양식 보호), 타겟 시스템 배정 -
10 커리큘럼 관리
curriculum		 완성 3-Screen 패턴, 트리 CRUD, 양식별 별칭, 깊이 설정, 분할 깊이 -
11 채점 시뮬레이션
scoring-sim		 완성 목록, 필터, 페이지네이션, 점수 입력 점수 입력 시 범위 검증(JS) 없음
12 상품·수신 관리
batch-receive		 부분 카드 UI, 브랜드별 그룹핑, 커리큘럼 연결 linkedCurIds 미설정 데이터 정합성 버그 (커리큘럼 생성 시 연결 누락)
13 시스템 설정
settings		 부분 6탭 구조, 공지사항 CRUD, 감사 로그 조회, PRD 배포 권한 admin 전용 회원 정보 수정·권한 편집이 스텁(alert만). RBAC 실제 제어 로직 없음

3. 핵심 이슈 (심각도순)

21건의 발견 이슈 중 상위 12건을 심각도순으로 정리합니다. 교육 서비스 특성상 휴먼 에러가 학습자 전체에 영향을 주므로, 배포·서비스 연결 관련 이슈를 최우선으로 배치했습니다.

Critical 롤백이 검수 없이 즉시 배포
배포 관리 (deploy) · L16126
현재 롤백은 confirm() 한 번으로 이전 버전 콘텐츠가 서비스에 즉시 교체됩니다. 검수 단계가 없어 오래된 버전으로 잘못 롤백하거나, 다른 콘텐츠의 롤백 버튼을 클릭하는 실수가 곧바로 서비스에 반영됩니다.
권고: 롤백 시 ① 사유 입력 필수 ② 롤백 대상 버전 미리보기 ③ 관리자 즉시 승인 또는 검수 후 배포 선택지 제공
Critical 미배포 상태에서 코드 연결 가능
과목코드 연결 (service-bind) · L16188
배포 완료되지 않은 콘텐츠에서도 코드 연결 관리 화면에 진입할 수 있습니다. 불완전하거나 검수 미통과 콘텐츠가 과목코드에 연결되면 학습자에게 잘못된 콘텐츠가 노출됩니다.
권고: 코드 연결 진입 시 해당 콘텐츠의 배포 상태를 체크하여, "배포 완료" 상태가 아니면 진입을 차단하고 안내 메시지 표시
High 배포 실행 버튼에 추가 확인 없음
배포 관리 (deploy) · L16027
배포 실행 버튼 클릭 시 단순 confirm()만 존재합니다. 다수 콘텐츠 일괄 배포 시 실수 클릭 한 번으로 검수되지 않은 콘텐츠까지 배포될 위험이 있습니다.
권고: 배포 실행 시 ① 배포 대상 목록 + 영향 범위 요약 모달 ② 배포 사유 입력 ③ "배포" 텍스트 직접 타이핑 확인 (GitHub 삭제 패턴)
High 스케줄 배포 삭제에 확인 없음
배포 관리 (deploy) · L16029
예약된 스케줄 배포를 삭제할 때 확인 과정이 없습니다. 실수로 삭제하면 예정된 배포가 누락됩니다.
권고: 삭제 시 "이 스케줄을 삭제하면 예약된 배포가 취소됩니다" 확인 다이얼로그 추가
High 강제 게시 버튼 이중 확인 부족
콘텐츠 등록 (content-register) · L6009
미완료 항목이 있는 상태에서 "강제 게시" 버튼을 누르면 confirm() 한 번으로 게시됩니다. 작성 중인 콘텐츠가 검수 요청 대기열에 올라갈 수 있습니다.
권고: 미완료 항목 목록을 명시한 경고 모달 + "강제 게시" 텍스트 입력 확인, 또는 미완료 시 강제 게시 자체를 비활성화
High 일괄 검수 요청 시 검수자 하드코딩
검수 관리 (review) · L6402
일괄 검수 요청 시 검수자가 항상 "김검수(QA팀)"로 고정됩니다. 실제 운영에서는 콘텐츠 유형·과목별로 다른 검수자를 지정해야 합니다.
권고: 검수자 선택 드롭다운/모달 추가. 커리큘럼 또는 과목별 기본 검수자 설정 기능 고려
High "반려"와 "수신 변경 무시"가 동일 함수 호출
커리큘럼 수신 (cur-receive) · L10445-10453
수신 목록에서 "반려"와 "변경 무시"가 같은 함수를 호출합니다. 운영자가 반려(사유 피드백 필요)와 무시(의도적 스킵)를 구분할 수 없어 수신 이력 추적이 어렵습니다.
권고: 반려 시 사유 입력 필수 + 수신 이력에 "반려"/"무시" 구분 기록. 반려는 발신 측에 알림, 무시는 내부 기록만
Medium 배포 완료 후 코드 연결 뷰 직접 링크 없음
배포 관리 (deploy) → 과목코드 연결 (service-bind)
배포 완료 후 자연스럽게 코드 연결 화면으로 이동해야 하지만, 현재 사이드바에서 별도로 찾아 들어가야 합니다. 2단계 안전 모델(배포 → 코드 연결)의 동선이 끊깁니다.
권고: 배포 완료 시 "코드 연결로 이동" CTA 버튼 또는 토스트 알림에 링크 포함
Medium 편집버전 생성 후 부분 갱신 혼란
커리큘럼 수신 (cur-receive) · L10494
편집버전 생성 시 체크된 항목만 "matched"로 갱신되고, 체크하지 않은 항목은 여전히 "검토필요"로 표시됩니다. 운영자가 일부만 처리한 건지, 버그인지 구분하기 어렵습니다.
권고: 미체크 항목에 대해 "건너뜀" 상태를 별도 표시하거나, 전체 처리 확인 후 일괄 갱신
Medium 작업 설정 범위 중복 체크 없음
작업 설정 (work-config) · L10730
동일한 커리큘럼 범위를 여러 작업에 중복 배정할 수 있습니다. 두 담당자가 같은 콘텐츠를 동시에 작업하면 충돌이 발생합니다.
권고: 범위 설정 시 기존 작업과 겹치는지 체크하여 경고 표시
Medium 작업 번호 생성 시 빈 항목 허용
콘텐츠 등록 (content-register) · L6908
작업 번호 자동 생성 시 필수 정보가 비어 있으면 "미지정"으로 채워집니다. 이후 추적이 어려워집니다.
권고: 필수 항목 미입력 시 작업 번호 생성을 차단하고, 어떤 항목이 비어 있는지 안내
Low 해답 편집기 미저장 이탈 경고 없음
콘텐츠 등록 (content-register)
해답 편집기에서 변경사항이 있을 때 다른 화면으로 이동해도 경고가 표시되지 않습니다.
권고: beforeunload 이벤트 또는 뷰 전환 시 변경 감지 로직 추가

4. 배포 프로세스 안전성 분석

Shawn이 특별히 우려한 영역입니다. "코드까지 결합 → 테스트 → 최종 배포" 순서와 "휴먼 에러 최소화"를 중심으로 현재 동선과 권고안을 비교합니다.

현재 동선 (위험 요소 포함)

콘텐츠 작업 (content-register)
↓
검수 요청 → 검수 완료 (review)
↓
배포 실행 — confirm() 1회로 즉시 배포
↓
코드 연결 — 미배포 콘텐츠도 연결 가능
↓
서비스 노출
롤백: confirm 1회 → 검수 없이 즉시 교체

권고 동선 (휴먼 에러 최소화)

콘텐츠 작업 (content-register)
↓
검수 요청 → 검수 완료 (review)
↓
배포 실행 — 대상 목록 확인 모달 + 사유 입력
↓
코드 연결 — 배포 완료 콘텐츠만 허용
↓
운영 시간 가드레일 → 서비스 노출
롤백: 사유 입력 → 검수 또는 관리자 즉시 승인

5. 미구현 영역

5-1. 로그인 / 인증 시스템

파일 전체(27,827라인)에 로그인 폼, 인증 토큰, 세션 관리 코드가 전혀 없습니다. 현재 프로토타입은 URL을 열면 바로 관리자 화면이 표시되는 상태입니다.

필요 구현: 로그인 페이지 UI, SSO 또는 자체 인증 연동, 세션 타임아웃 처리, 비밀번호 정책

※ Shawn도 이 부분이 빠져 있음을 인지하고 있음. 프로토타입 단계에서는 기능 흐름 검증에 집중하되, 실제 개발 시 최우선 구현 필요

5-2. 역할 기반 접근 제어 (RBAC)

시스템 설정에 "권한 관리" 탭이 존재하지만, 실제 제어 로직은 없습니다. 회원 편집·권한 변경 버튼을 누르면 alert만 표시됩니다. 현재 모든 사용자가 관리자(admin) 권한으로 모든 기능에 접근 가능합니다.

필요 구현:

역할접근 범위비고
콘텐츠 담당자콘텐츠 등록·수정, 자기 작업 범위만배포·코드 연결 접근 차단
검수자검수 관리, 반려/승인콘텐츠 수정 불가
배포 관리자배포 실행, 스케줄 관리PRD 배포는 admin만 (현재 구현됨)
시스템 관리자전체 (설정, 권한, 감사 로그)코드 연결, 롤백 포함

5-3. URL 기반 라우팅

현재 뷰 전환이 JavaScript 함수(switchView())로만 동작하여, 브라우저 뒤로가기가 작동하지 않습니다. 실제 운영 시 사용자 혼란을 야기합니다.

※ 실제 개발 시 프레임워크(React/Vue) 라우터로 자연 해결되는 영역. 프로토타입에서는 우선순위 낮음

5-4. 에러 바운더리

초기화 중 오류 발생 시 전체 앱이 중단됩니다. 미선언 변수 참조(L22727, L22866, L22782-22795)가 여러 곳에서 발견되었습니다.
