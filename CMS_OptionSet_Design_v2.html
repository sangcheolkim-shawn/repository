선택형 필드의 과목×유형 종속 문제 해결 — 양식 구조/옵션값 분리 아키텍처
CMS-111 2026-05-26 Shawn / 플랫폼기획팀 v2.0

v1.0 → v2.0 변경 사항 (개발자 피드백 반영)

수용
쟁점 1: 영향 필드 범위 확대 — v1에서 "12개 중 2개(17%)만 과목 종속"이라고 분석했으나, 실제 운영 데이터에서는 다수의 select 필드가 과목별로 다른 옵션을 가짐. 전수조사 필요로 변경.
확장
쟁점 2: 스코프 키 복합화 — v1의 scope_type = 'subject' 단일키에서, scope_type = 'subject_type' (과목 + 유형코드) 복합키로 확장. 같은 과목이라도 유형코드(ANS, CCV 등)에 따라 옵션이 다른 케이스 대응.
해소
쟁점 3: 워크플로우 타이밍 — "과목 바인딩 전에 옵션 조회 불가" 우려는 해소. "과목코드 연결"(step 4, 외부 서비스 매핑)과 "커리큘럼의 과목"(step 2, 커리큘럼 생성 시 확정)은 별개 개념. 콘텐츠 입력(step 3) 시점에 curriculum.subject + form.typeCode 모두 확정되어 있음.

목차

1. 문제 정의 2. 개발자 피드백 분석 (v2 신규) 3. 워크플로우 타이밍 검증 (v2 신규) 4. 현황 분석 — 과목 종속 필드 (v2 수정) 5. 방안 비교 6. 권장 방안: 옵션 세트 분리 (v2 수정) 7. DB 스키마 설계 (v2 수정) 8. API 인터페이스 (v2 수정) 9. 프론트엔드 구현 (v2 수정) 10. UI/UX 변경점 11. 마이그레이션 전략 12. 영향 범위 및 일정 (v2 수정)

1. 문제 정의

개발팀 피드백: "뉴드림스 입력양식은 과목×유형 수만큼 생성할 수밖에 없을 것 같다"

핵심 충돌

CMS 2.0 아키텍처의 핵심 원칙 "양식은 과목 독립"과, 현실의 "select 필드 옵션이 과목×유형마다 다르다"가 충돌합니다.

양식을 과목별 복제하면

  • 29개 과목 x 9개 양식 유형 = 261개 양식
  • 필드 1개 추가 시 261개 일괄 수정
  • "과목 독립" 아키텍처 원칙 붕괴
  • 운영자 관리 부담 폭증

옵션값만 분리하면

  • 양식 9개 유지 + 옵션 세트로 분기
  • 필드 변경 시 1곳만 수정
  • 아키텍처 원칙 100% 유지
  • 운영자는 옵션값만 관리

2. 개발자 피드백 분석 v2 신규

개발자의 3가지 핵심 지적과 대응 방향

피드백 원문

개발자:
"같은 과목이라도 유형코드에 따라 옵션값이 다릅니다.
ex) 영어 / 해답지(ANS), 핵심점검 Cover(CCV) → area_cd의 필드값이 다름

과목별 옵션형 필드의 옵션값을 따로 관리한다면 커리큘럼에 바인딩된 과목코드로 해당 옵션값을 조회할 수 있겠지만, 과목이 바인딩 되기 전까지는 해당하는 옵션형 필드의 옵션값을 조회할 방법이 없습니다.

CMS2.0의 컨셉인 입력양식 등록 → 커리큘럼 등록 → 콘텐츠 등록 → 과목 바인딩의 작업 흐름에서는 과목별 옵션값을 조회 불가능 할 것으로 보입니다."

쟁점별 분석

쟁점개발자 지적대응결과
1. 영향 필드 범위 area_cd만이 아니라 다수의 옵션형 필드가 과목별로 다름 v1의 "2개/17%" 분석은 프로토타입 기준. 운영 데이터 전수조사 필요 수용
2. 스코프 키 같은 과목이라도 유형코드(ANS, CCV)에 따라 옵션이 다름 스코프 키를 subject → subject + typeCode 복합키로 확장 확장
3. 워크플로우 타이밍 과목 바인딩 전에는 옵션 조회 불가 "과목코드 연결"과 "커리큘럼 과목"은 별개 → 다음 섹션에서 상세 검증 해소

스코프 키 확장 예시

같은 area 필드라도 (과목, 유형코드) 조합에 따라 옵션이 달라지는 실제 사례:

과목유형코드area_cd 옵션값scope_value
영어ANS (해답지)읽기, 쓰기, 듣기, 말하기E:ANS
영어CCV (핵심점검)어휘, 문법, 독해, 작문E:CCV
국어ANS (해답지)읽기, 쓰기, 문학, 문법K:ANS
수학ANS (해답지)대수, 기하, 통계, 해석M:ANS
v1 대비 변경: scope_value가 'M'(과목만)에서 'M:ANS'(과목:유형코드) 복합키로 변경됩니다. 이로써 같은 과목이라도 유형코드별로 다른 옵션 세트를 가질 수 있습니다.

3. 워크플로우 타이밍 검증 v2 신규

개발자 우려: "콘텐츠 입력 시점에 과목을 모르므로 옵션 조회 불가"에 대한 검증

3.1 두 가지 "과목"의 구분

커리큘럼의 과목 (step 2에서 확정)

"이 커리큘럼은 수학이다"
CURRICULUM_DATA.subject = '수학'
커리큘럼 생성 시 필수 입력 → 콘텐츠 입력 전에 이미 확정

과목코드 연결 (step 4에서 수행)

"이 콘텐츠를 뉴드림스 수학(M)에 보내자"
ncm_content_bindings.subject_code = 'M'
외부 서비스 매핑 → 콘텐츠 입력 이후

3.2 워크플로우별 데이터 가용성

단계작업subject 알 수 있나?typeCode 알 수 있나?옵션 조회 가능?
Step 1 입력 양식 등록 NO YES (양식의 typeCode) 불필요 (양식 구조 정의)
Step 2 커리큘럼 등록 YES (curriculum.subject) YES (바인딩된 forms[].typeCode) 가능하나 불필요
Step 3 콘텐츠 입력 YES YES 가능
Step 4 과목코드 연결 YES YES 가능 (이미 완료)

// 콘텐츠 편집기 진입 시 — step 3에서의 데이터 참조 체인
var workItem   = WORK_ITEMS_V7.find(w => w.id === selectedWI);
var curriculum = CURRICULUM_DATA.find(c => c.id === workItem.curriculumId);
var form       = FORM_TEMPLATES.find(f => f.id === workItem.formId);

var subject    = curriculum.subject;    // '수학' — step 2에서 이미 확정
var typeCode   = form.code.split('-')[0]; // 'ANS' — step 1에서 이미 확정

// 옵션 조회: subject + typeCode 복합키로 해석

6. 권장 방안: 옵션 세트 분리 v2 수정

양식 구조는 공유, 옵션값만 분기 — "Structure Sharing, Value Branching"

6.1 개념 모델 (v2)

양식 (Form)
→
필드 정의 (FieldDef)
→
옵션 그룹 참조
→
런타임: subject+typeCode로 세트 선택
→
드롭다운 옵션 렌더링

6.2 데이터 흐름 예시 (v2)

시나리오: "영어 기초 과정" 커리큘럼에서 "핵심점검(CCV) 양식"의 콘텐츠를 등록할 때

1. 편집기 진입 → 커리큘럼의 subject = '영어' 확인
2. 양식의 typeCode = 'CCV' 확인
3. area 필드 → optionGroupId = 'OG-AREA' 참조
4. OG-AREA에서 scopeValue = 'E:CCV' 세트 검색
5. 옵션 세트 ['어휘','문법','독해','작문'] 로드 → 드롭다운에 표시
6. 사용자는 "독해" 선택 → value = '독해' 저장

6.3 폴백 전략 (3단계)

1순위: 정확 매칭
E:CCV 세트
→
2순위: 과목만 매칭
E:_default 세트
→
3순위: 전체 기본
_default 세트
→
4순위: 필드 폴백
LT_FIELD_POOL.options
v2 변경: v1의 2단계 폴백에서 4단계로 확장. E:CCV → E:_default → _default → LT_FIELD_POOL.options 순으로 탐색합니다. 이를 통해 유형코드별 세트가 없어도 과목 기본 세트로 커버됩니다.

7. DB 스키마 설계 v2 수정

기존 ncm_ 네이밍 규칙 준수, 3개 테이블 — scope_type 복합키 지원

7.1 ERD (v2)

ncm_field_option_groups          ncm_field_option_sets          ncm_field_option_values
+-----------------------+     +------------------------+     +----------------------+
| id (PK)               |--+  | id (PK)                |--+  | id (PK)              |
| field_key    VARCHAR   |  |  | group_id (FK)          |  |  | set_id (FK)          |
| group_name   VARCHAR   |  +--| scope_value  VARCHAR   |  +--| value      VARCHAR   |
| scope_type   VARCHAR   |     | set_label    VARCHAR   |     | label      VARCHAR   |
| description  TEXT      |     | is_default   BOOLEAN   |     | sort_order INT       |
| created_at   TIMESTAMP |     | UNIQUE(group,scope)    |     | is_active  BOOLEAN   |
+-----------------------+     +------------------------+     +----------------------+
         1 : N                         1 : N

scope_type 값:
  'subject'       → scope_value = 'M', 'E', 'K'...
  'subject_type'  → scope_value = 'M:ANS', 'E:CCV', 'K:ANS'...   ← v2 추가
  'grade'         → scope_value = '중1', '초3'...                   (확장 예약)

7.2 DDL (v2)

-- 옵션 그룹: 과목+유형별 분기가 필요한 select 필드 선언
CREATE TABLE ncm_field_option_groups (
    id           SERIAL PRIMARY KEY,
    field_key    VARCHAR(50)  NOT NULL,
    group_name   VARCHAR(100) NOT NULL,
    scope_type   VARCHAR(20)  NOT NULL DEFAULT 'subject_type',
    -- v2: 'subject_type'이 기본값 (과목+유형코드 복합키)
    -- 'subject': 과목만으로 분기 (유형코드 무관한 경우)
    -- 'grade': 학년으로 분기 (향후 확장)
    description  TEXT,
    created_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    UNIQUE(field_key)
);

-- 옵션 세트: scope 기준값별 옵션 컨테이너
CREATE TABLE ncm_field_option_sets (
    id           SERIAL PRIMARY KEY,
    group_id     INT          NOT NULL REFERENCES ncm_field_option_groups(id) ON DELETE CASCADE,
    scope_value  VARCHAR(50)  NOT NULL,
    -- v2: 복합키 형식 'M:ANS', 'E:CCV' 또는 과목만 'E:_default'
    set_label    VARCHAR(100),
    is_default   BOOLEAN      NOT NULL DEFAULT FALSE,
    created_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    UNIQUE(group_id, scope_value)
);

-- 옵션 값: 실제 드롭다운 항목
CREATE TABLE ncm_field_option_values (
    id           SERIAL PRIMARY KEY,
    set_id       INT          NOT NULL REFERENCES ncm_field_option_sets(id) ON DELETE CASCADE,
    value        VARCHAR(200) NOT NULL,
    label        VARCHAR(200),
    sort_order   INT          NOT NULL DEFAULT 0,
    is_active    BOOLEAN      NOT NULL DEFAULT TRUE
);

9.2 런타임 해석 헬퍼 (v2)

/**
 * 과목+유형코드 컨텍스트에 따른 select 옵션 해석 (v2)
 * @param {string} fieldKey   - LT_FIELD_POOL의 필드 key (예: 'area')
 * @param {string} subject    - 커리큘럼의 과목 (예: 'E' = 영어)
 * @param {string} typeCode   - 양식의 유형코드 (예: 'CCV' = 핵심점검)
 * @returns {string[]}        - 옵션 배열
 */
function _resolveFieldOptions(fieldKey, subject, typeCode) {
  var group = (OPTION_GROUPS || []).find(function(g) {
    return g.fieldKey === fieldKey;
  });
  if (!group) {
    // 옵션 그룹 없으면 LT_FIELD_POOL 기본값 폴백
    var pf = LT_FIELD_POOL.find(function(f) { return f.key === fieldKey; });
    return pf ? (pf.options || []) : [];
  }
  var sets = group.sets;
  var set = null;

  if (group.scopeType === 'subject_type') {
    // 1순위: subject:typeCode 정확 매칭
    var exactKey = subject + ':' + typeCode;
    set = sets.find(function(s) { return s.scopeValue === exactKey; });
    // 2순위: subject:_default (과목 기본)
    if (!set) {
      var subDefault = subject + ':_default';
      set = sets.find(function(s) { return s.scopeValue === subDefault; });
    }
  } else {
    // scope_type = 'subject' (v1 호환)
    set = sets.find(function(s) { return s.scopeValue === subject; });
  }
  // 3순위: 전체 기본
  if (!set) set = sets.find(function(s) { return s.scopeValue === '_default'; });
  return set ? set.values : [];
}
