2-2

콘텐츠 관리 구조의 비효율성

과목코드 x 콘텐츠 1:1 바인딩의 한계

AS-IS 현행: 1:1 구조

과목코드 AM — 콘텐츠 A 복제
과목코드 AM2 — 콘텐츠 A 복제
과목코드 AM3 — 콘텐츠 A 복제
... AM4, AM5, AMA, AMA2, AMB ...
681코드 x 4~5시스템 = 곱셈 증가
동일 데이터가 코드 수만큼 복제, 관리 부담 기하급수적 증가
VS

TO-BE 신규: 1:N 구조

스피드수학 커리큘럼 -> AM 써밋스피드수학
-> AM2 써밋스피드수학2
-> AM3 써밋스피드수학3
-> AM4 써밋스피드수학4
-> AM5 써밋스피드수학5
-> AMA 써밋스피드수학A
-> AMA2 써밋스피드수학A2
-> AMB 써밋스피드수학B
1 커리큘럼 → N 과목코드 바인딩
콘텐츠는 하나, 바인딩 설정만으로 8개 과목코드에 연결
