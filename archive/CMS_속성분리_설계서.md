# CMS 콘텐츠 속성 분리 설계서

## 1. 현재 구조의 문제

현재 CMS는 19개 콘텐츠 유형(AN2, CBT, FLC, VKS 등)에 대해 모든 속성을 DB 컬럼으로 관리하고 있다.
작업자가 콘텐츠를 등록할 때 해당 유형의 모든 필드가 한 화면에 나열되며, 시스템이 요구하는 필드와 운영상 필요한 메모/맥락 정보가 구분 없이 섞여 있다.

**핵심 문제**: 시스템 필수 정보와 운영 맥락 정보의 경계가 없다.

---

## 2. 분리 원칙

```
┌─────────────────────────────────────────────┐
│              콘텐츠 1건                       │
├──────────────┬──────────────────────────────┤
│  고정 필드    │  자유 컨텍스트               │
│  (System)    │  (Context)                   │
├──────────────┼──────────────────────────────┤
│ • DB 컬럼     │ • 자유 텍스트 메모            │
│ • 시스템 연동  │ • 태그 (키워드)              │
│   필수 값     │ • 작업 히스토리 노트          │
│ • 파일 메타   │ • 참고 링크/첨부             │
│              │ • 담당자 간 커뮤니케이션       │
├──────────────┼──────────────────────────────┤
│ 수정 제한     │ 자유 편집                    │
│ 검증 필수     │ 검증 없음                    │
│ 시스템 연동   │ CMS 내부 전용               │
└──────────────┴──────────────────────────────┘
```

### 고정 필드 (System Fields)
- 외부 시스템(뉴드림스, 눈높이365, 성장판, 학습관)이 요구하는 필드
- DB_COLUMNS에 정의된 유형별 컬럼 (AN2: 문항번호/정답/배점/비고 등)
- FILE_ACCEPT에 정의된 파일 업로드 규격
- **변경 시 검증 필수**, 배포 시 이 필드만 외부 전송

### 자유 컨텍스트 (Context Layer)
- 작업자가 자유롭게 추가하는 운영 정보
- 형식 제한 없음 (텍스트, 태그, 링크 등)
- **검색/필터링 가능**하되 외부 시스템에는 전송하지 않음
- 콘텐츠의 생명주기 내내 축적되는 "맥락 기록"

## 3. 자유 컨텍스트 데이터 구조

```javascript
// 콘텐츠 아이템별 컨텍스트
const CONTEXT_STORE = {
  // key = 콘텐츠 식별자 (예: wf1_FLC_s1)
  'wf1_FLC_s1': {
    // 자유 메모 (시간순 축적)
    notes: [
      { id: 'n1', text: '원본 파일 해상도 확인 필요 — 디자인팀 이영희 확인 중',
        author: '홍길동', at: '2026-03-10 14:30', pinned: true },
      { id: 'n2', text: '3회차 이미지 교체 완료. 이전 버전은 /archive/v1 참고',
        author: '김철수', at: '2026-03-11 09:15', pinned: false },
    ],
    // 태그 (자유 키워드)
    tags: ['긴급', '이미지교체', '디자인확인'],
    // 참고 링크
    links: [
      { label: '원본 디자인 파일', url: 'https://figma.com/...' },
      { label: '기획서 3.2절', url: '/docs/planning_v3.pdf#page=12' },
    ],
    // 마지막 수정
    updatedAt: '2026-03-11 09:15',
  }
};
```

## 4. UI 설계 — 콘텐츠 등록 화면 변경

### Before (현재)
```
┌─────────────────────────────────────┐
│ [AN2 채점] NE1 / 1세트              │
├─────────────────────────────────────┤
│ 문항번호  │ 정답  │ 배점 │ 비고     │  ← 전부 동일 레벨
│ ───────────────────────────────────│
│ 1        │ ③    │ 5   │ ...      │
│ 2        │ ①    │ 5   │ ...      │
│ ...                                │
│                                     │
│ (빈 필드가 많고 채워야 할 것이 많음)    │
└─────────────────────────────────────┘
```

### After (개선)
```
┌─────────────────────────────────────────────────────┐
│ [AN2 채점] NE1 / 1세트                               │
├──────────────────────────┬──────────────────────────┤
│ 📋 시스템 필수 필드       │ 📝 컨텍스트              │
│ (뉴드림스 연동)           │                          │
├──────────────────────────┤                          │
│ 문항번호 │ 정답 │ 배점    │ [메모]                   │
│ ─────────────────────── │ 원본 파일 해상도 확인 필요  │
│ 1       │ ③   │ 5      │ — 홍길동, 3/10 14:30     │
│ 2       │ ①   │ 5      │                          │
│ ...                      │ 3회차 이미지 교체 완료     │
│                          │ — 김철수, 3/11 09:15     │
│ ✅ 3/6 입력완료           │                          │
├──────────────────────────┤ [태그]                   │
│ 💾 저장  │ 📤 검수요청    │ #긴급 #이미지교체         │
│                          │                          │
│                          │ [+ 메모 추가]             │
└──────────────────────────┴──────────────────────────┘
```

**핵심 변경점:**
1. 좌측은 시스템이 요구하는 고정 필드 — "이것만 채우면 됩니다"
2. 우측은 자유 컨텍스트 패널 — 접었다 펼 수 있음
3. 메모는 시간순으로 축적 — 작업 히스토리 역할
4. 태그로 콘텐츠 간 연관 관계 형성 (같은 태그끼리 검색)

## 5. 컨텍스트 활용 시나리오

### 시나리오 A: 작업자 간 인수인계
홍길동이 작업하다 김철수에게 넘길 때, 별도 메신저 없이 컨텍스트 메모에 상황 기록.
김철수가 해당 콘텐츠를 열면 이전 맥락이 즉시 보임.

### 시나리오 B: 검수자의 피드백
검수 반려 시 "어디가 문제인지"를 컨텍스트에 기록.
작업자가 수정 후 "수정 완료" 메모 추가 → 검수자가 이력 확인.

### 시나리오 C: 운영 태그로 일괄 관리
'#이미지교체' 태그가 붙은 콘텐츠를 한번에 검색 →
이번 달 이미지 교체 작업 전체 현황 파악.

### 시나리오 D: 참고 자료 연결
기획서, 디자인 파일, 이전 버전 등을 링크로 연결 →
"이 콘텐츠가 왜 이렇게 만들어졌는지" 맥락 추적 가능.

## 6. 기존 DB_COLUMNS과의 관계

| 유형 | 고정 필드 (DB_COLUMNS) | 자유 컨텍스트 |
|------|----------------------|--------------|
| AN2 (채점) | 문항번호, 정답, 배점, 비고 | 메모, 태그, 링크 |
| CBT (핵심점검) | 섹션, 본문, 핵심어 | 메모, 태그, 링크 |
| FLC (플래시카드) | 파일 업로드 (.jpg/.png/.pdf) | 메모, 태그, 링크 |
| VKS (동요) | 파일 업로드 (.mp3/.wav) | 메모, 태그, 링크 |

**고정 필드는 유형마다 다르지만, 자유 컨텍스트는 모든 유형에 동일하게 적용.**

## 7. 검색/필터 확장

현재 필터: 과목, 단계, 상태, 담당자
추가 가능: **태그 검색**, **메모 내 텍스트 검색**, **핀 고정 메모만 보기**

```
검색바: [콘텐츠명 또는 태그로 검색...]  [#긴급] [#이미지교체] [#확인필요]
```

태그는 사용자가 자유롭게 생성하되, 자주 사용되는 태그는 자동완성으로 제안.
