# CMS 유형 코드 레지스트리

> **관리 주체**: 개발자 / PI (시스템 담당자)
> **최종 갱신**: 2026-04-20
> **용도**: 입력 양식 관리 메뉴에서 양식에 부여하는 "유형 코드"의 마스터 목록. 레거시 시스템의 기존 유형을 그대로 승계하며, 신규 양식 생성 시 이 목록을 참조하여 코드를 지정한다.

## 2. 현재 양식 ↔ 유형 코드 매핑 (FORM_TEMPLATES — 8종)

입력 양식 관리에서 등록된 양식과 부여된 유형 코드의 현행 매핑이다.

| 양식 ID | 양식명 | 유형 코드 | 상태 | 주요 필드 |
|---------|--------|-----------|------|-----------|
| FORM-ANS-A | 정답·해설 입력 양식 | `ANS-A` | 활성 | 문항번호, 정답, 배점, 해설, 비고 |
| FORM-AN2-A | 자동채점 입력 양식 | `AN2-A` | 활성 | 문항번호, 정답, 배점, 난이도 |
| FORM-AN3-A | 서술형 채점 양식 | `AN3-A` | 활성 | 문항, 키워드, 모범답안, 부분점수, 채점기준, 분량제한 |
| FORM-FLC-A | 플래시카드 입력 양식 | `FLC-A` | 활성 | 앞면, 뒷면, 이미지, 음성 |
| FORM-STDG-NH | 영상강의 입력 양식 | `STDG-NH` | 활성 | 강의명, 영상파일, 강의시간, 썸네일, DRM, 비고 |
| FORM-CBT-A | 핵심점검 입력 양식 | `CBT-A` | 활성 | 문항, 보기, 정답, 해설, 난이도 |
| FORM-EVA-A | 평가 문제지 양식 | `EVA-A` | 초안 | 평가명, 문항집, 배점배분, 제한시간, 채점방식 |
| FORM-WEB-A | 웹 학습콘텐츠 양식 | `WEB-A` | 초안 | 페이지명, HTML콘텐츠, 첨부파일 |

### 유형 코드 네이밍 규칙

```
{유형코드}-{변형}
예: ANS-A, AN2-A, STDG-NH, CBT-A
```

- **유형코드**: CONTENT_TYPES의 코드를 그대로 사용 (ANS, AN2, CBT 등)
- **변형(Variant)**: 같은 유형에 대해 서비스·목적별 양식이 다를 때 구분자 (A=기본, NH=눈높이 전용 등)
- **선택사항**: 유형 코드가 없는 양식도 허용 — API 연동 없이 운영하는 경우

## 5. 유형 코드 운영 규칙

### 레거시 승계
- 기존 CONTENT_TYPES 19종은 **코드 변경 없이 그대로 승계**
- 과목별 유형 매핑(SUBJECT_TYPE_MAP)도 현행 유지
- 신규 CMS의 양식에 유형 코드 부여 시 이 목록의 코드를 사용

### 신규 유형 코드 추가
- 개발자/PI가 협의하여 결정
- 코드 형식: 영문 대문자 2~4자 (예: `ANS`, `CBT`, `STDG`)
- CONTENT_TYPES에 등록 후 SUBJECT_TYPE_MAP에 과목별 매핑 추가

### 양식 유형 코드 (FORM_TEMPLATES.code)
- 양식 생성 시 **선택사항** — 개발자가 지정
- 형식: `{유형코드}-{변형}` (예: `ANS-A`, `STDG-NH`)
- API 연동이 필요한 양식만 코드 부여, 나머지는 빈값 허용
- 유형 코드의 앞부분(하이픈 전)이 CONTENT_TYPES 코드와 매칭됨

### DB 구조
```
ncm_type_codes (code, label, group_name, icon, color, bg_color, sort_order)
ncm_subject_type_map (subject_code, type_code)
ncm_form_templates (id, name, type_code, status)  -- type_code = nullable
```
