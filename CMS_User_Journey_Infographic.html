CMS User Journey Map — 인포그래픽

콘텐츠 업무 흐름 3가지 시나리오 + 권한 관리 설계 (도형 기반 시각화)

프로젝트: CMS 고도화 (NDCM) 프로토타입: v8.5 작성일: 2026-05-19 작성자: Shawn (플랫폼기획팀)

3가지 시나리오 전체 흐름

CMS 콘텐츠 업무의 핵심 시나리오 3가지를 한눈에 비교합니다. 신규 등록(11단계), 사용 전 수정(7단계), 사용 중 수정(9단계+롤백)의 전체 흐름을 색상 코딩으로 구분합니다.

Journey 1 — 신규 콘텐츠 등록 상세 플로우

콘텐츠가 존재하지 않는 상태에서 양식 정의, 커리큘럼 설계, 콘텐츠 제작, 검수, 과목코드 연결, 배포까지 전체 워크플로우 11단계를 Phase별로 추적합니다.

Journey 2 — 콘텐츠 수정 (사용 전) 상세 플로우

아직 서비스에 배포되지 않은 콘텐츠를 수정하는 워크플로우입니다. 기존 작업 항목을 찾아 직접 수정(덮어쓰기) 후 재검수→배포 프로세스를 거칩니다. 버전 관리 없이 원본을 직접 수정합니다.

Journey 3 — 콘텐츠 수정 (사용 중) 상세 플로우

이미 서비스에 배포되어 운영 중인 콘텐츠를 수정하는 워크플로우입니다. 운영 중인 콘텐츠는 직접 수정할 수 없으며, 개정(revision) 생성을 통해 새 버전을 만들고 수정→검수→재배포 과정을 거칩니다. 문제 발생 시 롤백이 가능합니다.

CMS 역할 기반 접근 제어 (RBAC)

3가지 Journey에서 도출된 4개 역할과 12개 화면에 대한 접근 권한을 시각화합니다. CRUD(전체), R(읽기), OWN(본인 데이터만), 제한(특정 액션만)으로 구분합니다.
