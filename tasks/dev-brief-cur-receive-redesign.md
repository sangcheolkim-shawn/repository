# [기획 변경] 커리큘럼 수신 목록 — 처리 상태 중심 리디자인

**관련 이슈**: CMS-132  
**변경일**: 2026-06-09  
**프로토타입**: cms_stg_prd_release_v1.1.html  

> **한 줄 요약**: 커리큘럼 수신 목록에서 파이프라인 네비게이션(코드연결→수신→콘텐츠→배포)을 제거하고, 수신 건의 **처리 상태**(조치필요/작업중/보류/완료)를 중심으로 테이블 형태 목록을 재구성함.

---

## 1. 변경 배경

기존 커리큘럼 수신 화면 상단에 "코드연결 → 수신 → 콘텐츠 → 배포" 파이프라인 네비게이션이 있었음.
이 동선은 메뉴 간 이동을 유발하여 "이 메뉴에서 처리해야 하는 역할"이 불분명해지는 문제가 있었음.

**변경 원칙**: 각 메뉴는 자기 역할에 집중. 커리큘럼 수신 메뉴는 **수신된 변경사항의 처리 현황 추적**만 담당.

## 3. 처리 상태 판정 로직 (핵심)

각 커리큘럼 그룹(linkedCurId 기준 묶음)의 처리 상태를 아래 로직으로 자동 판정함.

```
function _crGetGroupProcessState(group):
  
  1. status = 'unlinked' → "미연결" (별도 분류, 완료 탭에 포함)
  2. status = 'synced' 또는 'rejected' → "완료"
  3. status = 'holding' → "보류"
  4. status = 'detected' 이면:
     4-1. CURRICULUM_DATA에서 status='temp' AND sourceId = group.curId 인 커리큘럼 존재?
          → 존재함: "작업중" (편집버전이 만들어져서 작업 진행 중)
          → 없음:   "조치필요" (변경 감지됨, 아직 편집버전 미생성)
```

### DB 관점 구현 포인트

| 처리 상태 | 판정 조건 (API 레벨) | 색상 |
|-----------|---------------------|------|
| **조치필요** | `ncm_cur_receives.status = 'detected'` AND `ncm_curricula`에 해당 커리큘럼의 temp 없음 | #E65100 (오렌지) |
| **작업중** | `ncm_cur_receives.status = 'detected'` AND `ncm_curricula`에 `status='temp', source_id=linked_cur_id` 존재 | #0055B8 (블루) |
| **보류** | `ncm_cur_receives.status = 'holding'` | #1565C0 (인디고) |
| **완료** | `ncm_cur_receives.status IN ('synced', 'rejected')` | #2E7D32 (그린) |
| **미연결** | `ncm_cur_receives.linked_cur_id IS NULL` | #86868B (그레이) |

### "작업중" 상태의 작업 진행률 표시

편집버전(Temp)이 존재하면 해당 Temp 커리큘럼 ID로 `ncm_work_assignments`를 조회하여 진행률 표시:
- 진행률 = 완료 건수 / 전체 건수 (프로그레스 바 + "N/M" 텍스트)

## 4. 테이블 컬럼 명세

| # | 컬럼명 | 데이터 소스 | 표시 형태 |
|---|--------|-------------|-----------|
| 1 | **커리큘럼** | `linked_cur_name` + 과목코드 칩 (최대 3개, 초과 시 +N) + 버전 정보 | 커리큘럼명(굵은체) + 코드칩 서브텍스트 + "코드 N건 · v3 → v4" |
| 2 | **변경내용** | `change_type` + `diff_summary` | 타입 라벨(구조 변경/콘텐츠 변경/레이블 변경) + 요약 텍스트 |
| 3 | **수신일** | `detected_date` | 날짜(MM-DD) + 경과일 배지 (5일↑ 빨강, 3일↑ 노랑, 이하 파랑) |
| 4 | **편집버전** | `ncm_curricula` Temp 존재 여부 | "vN 편집중"(파랑) / "미생성"(오렌지 또는 회색) |
| 5 | **작업** | `ncm_work_assignments` 진행률 | 프로그레스 바 + "완료/전체" (Temp 없으면 "—") |
| 6 | **처리상태** | 판정 로직 결과 | 상태 배지 (조치필요/작업중/보류/완료/미연결) |

행 클릭 시 기존 상세 뷰(`_crOpenDetail`)로 진입. 완료/미연결 행은 `opacity: 0.6` 처리.

## 5. 필터 / 소트 / 검색 명세

### 검색
- 대상: 커리큘럼명(`linked_cur_name`) + 과목코드(`subject_code`) + 과목명(`subject_name`)
- 실시간 필터링 (입력 시 즉시 반영)

### 과목코드 필터
- 수신 데이터 전체에서 고유 과목코드를 추출하여 드롭다운 옵션 생성
- "전체" 기본값 + 개별 코드 선택

### 소트 옵션 (3종)
| 값 | 라벨 | 정렬 기준 |
|----|------|-----------|
| `date-desc` | 수신일 최신순 | `detected_date DESC` |
| `elapsed-desc` | 경과일 많은순 | `NOW() - detected_date DESC` |
| `no-temp-first` | 편집버전 미생성 우선 | Temp 없는 그룹 먼저, 동일 시 수신일 최신순 |

## 6. 데이터 모델 (변경 없음)

기존 `ncm_cur_receives` 테이블 구조를 그대로 사용.
목록 API에서 아래 정보를 JOIN/서브쿼리로 제공해야 함:

| 추가 조회 필요 항목 | 소스 | 용도 |
|---------------------|------|------|
| Temp 커리큘럼 존재 여부 | `ncm_curricula WHERE status='temp' AND source_id = linked_cur_id` | 처리 상태 판정 + 편집버전 컬럼 |
| Temp의 version 값 | 같은 row | "vN 편집중" 표시 |
| 작업 진행률 | `ncm_work_assignments WHERE curriculum_id = temp_cur_id` | 작업 컬럼 (완료/전체) |
