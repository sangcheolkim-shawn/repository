# 데모 시나리오 점검 체크리스트

> 프로토타입 변경 후 반드시 이 문서의 해당 뷰 체크리스트를 확인할 것.
> 변경 관리 프로토콜(CLAUDE.md) 참조.
> 최종 업데이트: 2026-06-10

---

## 사용법

1. 프로토타입 변경 작업 완료 후, 변경된 뷰의 체크리스트를 실행
2. 각 시나리오가 정상 동작하는지 확인 (샘플 데이터 + UI 렌더링)
3. ⚠️ 표시 항목은 특히 주의 — 이전 데모에서 문제가 된 사례
4. 결과를 Shawn에게 보고: "✅ N개 정상 / ⚠️ M개 확인 필요"

---

## 뷰별 체크리스트

### 1. 과목코드 연결 (service-bind) ⚠️ 최근 변경

**필수 데모 시나리오:**
- [ ] 상단 요약 카드 4칸이 정확한 수치 표시 (커리큘럼 연결/미연결/양식 연결/시스템별)
- [ ] 연결 완료 커리큘럼 카드: 양식별 미니 뱃지에 과목코드 + 작업단계 표시
- [ ] 미연결 커리큘럼 카드: 빨간 테두리 + "미연결 N" 뱃지 + "연결하기" 버튼
- [ ] 과목 필터(전체/수학/영어 등) 동작 확인
- [ ] 시스템 필터(전체/ND/NH/SJ/NH365) 동작 확인
- [ ] 정렬(이름순/연결률순/미연결 우선) 동작 확인
- [ ] 검색: 커리큘럼명, 과목코드로 검색 동작
- [ ] "코드 추가" 모달 → 바인딩 → 뱃지 반영
- [ ] 카드 하단 점프: "커리큘럼 보기" → curriculum 뷰 이동
- [ ] 카드 하단 점프: "작업 설정" → work-config 뷰 이동

**데이터 요구사항:**
- 연결 완료 커리큘럼 최소 2개 (다양한 시스템: ND, NH)
- 미연결 커리큘럼 최소 2개 (양식은 있으나 코드 미연결)
- 양식 0개인 커리큘럼 1개 ("양식 없음" 표시 확인)
- 부분 연결 커리큘럼 1개 (3양식 중 2개만 연결)
- 작업 단계 다양성: 등록/검수/배포 단계 섞여 있을 것

**점프 연결 확인:**
- `_jumpToCurriculum(curId)` → 커리큘럼 편집기로 이동
- `_jumpToWorkConfig(curId)` → 작업설정 뷰로 이동
- `_jumpToForm(formId)` → 양식관리 뷰로 이동

### 2. 커리큘럼 수신 (cur-receive) ⚠️ 최근 구현

**필수 데모 시나리오:**
- [ ] 목록 뷰: 커리큘럼 그룹핑(linkedCurId 기준) 정상 표시
- [ ] 수신 상태별 카운트 정확 (감지됨/보류/반영완료/반려/미연결)
- [ ] 상세 뷰 진입: 카드 클릭 → 3-STEP 의사결정 흐름 표시
- [ ] STEP1: 버전 비교 (현재 vs 수신) — diff 하이라이팅 정상
- [ ] STEP2: 영향 분석 — 연결된 과목코드, 작업 현황 표시
- [ ] STEP3: 판정 — "수용/보류/반려" 버튼 동작
- [ ] 편집버전 생성: "Temp(편집버전) 생성" → 커리큘럼 편집기 이동
- [ ] 목록으로 돌아가기 (`_crBackToList()`) 정상 동작
- [ ] 파이프라인 카운트 (`_crGetPipelineCounts()`) 정확

**데이터 요구사항:**
- 감지됨(detected) 상태 수신 건 최소 2개
- 보류(holding) 상태 1개
- 반영완료(synced) 상태 1개
- 연결된 커리큘럼이 있는 수신 건 + 미연결(unlinked) 수신 건
- diff가 있는 수신 건 (detectedVer, diffDetail[] 데이터)
- 편집버전이 이미 존재하는 커리큐럼 1개 (중복 생성 방지 확인)

**점프 연결 확인:**
- `_crOpenDetail(crId)` → 상세 뷰 정상 진입
- `_crCreateTemp(curId, newVer)` → 커리큘럼 편집기 이동
- `_crBackToList()` → 목록 복귀

### 3. 커리큘럼 관리 (curriculum) ⚠️ 빌더 변경

**필수 데모 시나리오:**
- [ ] 커리큘럼 목록: 활성/초안/편집버전 구분 표시
- [ ] 2컬럼 빌더 레이아웃: 좌측 트리 + 우측 편집 영역
- [ ] 트리 빌더: depth 계층 구조 펼침/접기
- [ ] 양식 배치: splitDepth에 양식 연결 표시
- [ ] 양식 검색 팝업: 양식 추가/제거
- [ ] 버전업: 활성 → 편집버전 생성
- [ ] depth 추가/삭제/이름 변경
- [ ] 메타 스키마(태그) 설정

**데이터 요구사항:**
- 활성(active) 커리큘럼 최소 3개 (다양한 depth 구조)
- 초안(draft) 커리큘럼 1개
- 편집버전(temp) 커리큐럼 1개 (sourceId 연결)
- 3단계 이상 depth 커리큘럼 1개
- 양식 2개 이상 배치된 커리큘럼 1개
- 메타 스키마 설정된 커리큘럼 1개

**점프 연결 확인:**
- `_jumpToCurriculum(curId)` → 편집기 진입
- `_jumpToForm(formId)` → 양식관리 이동
- `_jumpToWorkConfig(curId)` → 작업설정 이동

### 4. 작업 진행 보드 (workflow)

**필수 데모 시나리오:**
- [ ] 칸반 4단계 표시 (배정→등록→검수→배포)
- [ ] 카드 드래그&드롭 (단계 이동)
- [ ] 카드 클릭 → 상세 슬라이드
- [ ] 필터: 과목/시스템/담당자
- [ ] 검색 동작

**데이터 요구사항:**
- 각 단계에 카드 최소 2개
- 다양한 시스템(ND/NH) 카드
- 진행률 0%~100% 카드 섞여 있을 것

## 변경 영향 매트릭스

뷰 간 데이터 의존성. 한 뷰를 변경하면 아래 뷰들도 점검 필요.

| 변경 뷰 | 영향받는 뷰 | 공유 데이터 |
|---------|------------|------------|
| **curriculum** | service-bind, work-config, cur-receive, service-content | CURRICULUM_DATA, _curGetAllForms |
| **service-bind** | service-content, deploy | SVC_CURRICULA, boundCodes |
| **cur-receive** | curriculum (편집버전 생성) | CUR_RECEIVE_DATA, _crCreateTemp |
| **form-manage** | curriculum, work-config, service-bind | FORM_TEMPLATES |
| **work-config** | workflow, content-register, review | WORK_ITEMS_V7 |
| **workflow** | content-register, review, deploy | WORK_ITEMS_V7 (phase/status) |

## 공용 헬퍼 영향 목록

| 헬퍼 | 사용 뷰 | 변경 시 주의 |
|------|---------|------------|
| `_renderCurCardTop()` | service-bind, work-config, service-content, cur-receive | UI 구조 변경 시 모든 사용 뷰 점검 |
| `_curGetAllForms()` | curriculum, service-bind, cur-receive | 양식 수집 로직 변경 시 연결 카운트 확인 |
| `_isCurBound()` / `_getCurBoundCodes()` | service-bind, service-content, cur-receive | 연결 판정 로직 변경 시 전체 점검 |
| `_syncWiToWorkflow()` | workflow, work-config | 데이터 브릿지 변경 시 칸반 확인 |
| `_jumpTo*()` | 전 뷰 | 점프 대상 뷰가 정상 렌더링되는지 확인 |

## 교훈 기록

### 2026-06-10: PI 회의 데모 실패
- **원인**: 과목코드 연결·커리큐럼 수신·빌더 뷰의 UI가 이전 버전 대비 크게 변경되었으나, 데모 시나리오와 샘플 데이터가 변경된 UI에 맞게 갱신되지 않음
- **증상**: 회의 중 설명 시 화면이 예상과 다르게 표시되어 발표 중단
- **대책**: 변경 관리 프로토콜 도입 (CLAUDE.md) + 데모 시나리오 점검 체크리스트 (이 문서)
- **규칙**: 레이아웃·정보구조 수준의 변경 시 반드시 사전 알림 + 시나리오 점검 후 배포
